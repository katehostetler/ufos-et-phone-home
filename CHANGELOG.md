## 2026-05-09

### Globe look + UFOs + HUD polish (even later 5/9 session)
- **Co-located pins fan out** — records that shared a location (e.g. the two Gemini 7 records both at "Low Earth Orbit", the five at the Strait of Hormuz) used to stack into one overlapping pin, so whichever drew last hid the others (it looked like the LEO document pin was red — that was the audio "video" pin on top of it). They're now spread into a small ring around the shared point (`_pinLat`/`_pinLng` on the pin data; the records' real `location` is untouched, so clicking still gathers *all* records at that place). The PinRail/queue beacon flies to and lights up the fanned-out pin.
- **A tiny saucer loops the Moon** — `LunarMoon.tsx` now has a little brushed-silver flying saucer doing laps around the orbiting Moon. It's *super* skittish: bring your cursor anywhere near it (in screen space) and it bolts — whips around the Moon at ~12× speed, sometimes flips direction, jitters — so you basically can't catch it. Decorative; respects reduced-motion.
- **Starfield fixed** — the 3D star points were rendering *on top of* the globe and ballooning into big white squares when you zoomed out. They're now drawn first (`renderOrder -1`) with `depthTest`/`depthWrite` off so the opaque globe always paints over them (stars only ever show in the black space *around* the globe), and `sizeAttenuation` is off so they stay a fixed pixel size at any zoom. Three layers (bright/medium/faint) with brightness baked into the per-vertex colour.
- **Globe / city lights** — kept the globe as bright as before (the canvas filter is no longer dimmed); instead the shader patch now tones the *city lights* down (~20% dimmer + 30% desaturated, so they read as soft sodium-amber, not the vivid pin gold) AND lifts the dark ocean / unlit land toward a deep navy instead of pure black. And the **city-light shimmer actually works now** — the old patch keyed off `vMapUv` (fragile); it's been rewritten to seed the twinkle from each pixel's own brightness (no varying dependency), with the intensity bumped up so you can see it.
- **UFOs — always ≥2, always different sizes, always different speeds.** `UFO_POOL` now has a "small + fast" half (scale ~0.6–0.78, cruise 4.6–6) and a "large + slow" half (scale ~1.35–1.65, cruise 1.7–2.4). `makeSpawnManager` has a `minActive` floor (default 2) that's refilled immediately (no cooldown), and when it spawns the second craft it picks the *opposite size class* — so there's always a clearly-bigger-slower one and a smaller-faster one on screen, up to a cap of 4. Each saucer's invisible hit-sphere is now ~5 world units regardless of its visual size (small fast ones aren't extra-hard to catch).
- **PinRail "you're being flown here" beacon** — when the slim pin browser (or the dock coverflow) flies the globe to a record, that pushpin now lights up as a grown white beacon (not a media-type colour) so you can tell which pin it moved you to even in a dense cluster. Cleared when the rail/queue closes.
- **Removed the hover-"jump"** on pushpins — toggling a clustered pin's scale as the raycaster flipped between neighbours made pins flicker/"jump everywhere"; hover feedback is just the tooltip now (the beacon is reserved for the PinRail cue).
- **Hover tooltip** is outlined in the record's file-type colour (red / cyan / gold) and carries the file-type glyph (`▶` / `⊡` / `▤`).
- **Moon click** pauses the globe's auto-rotation while you read its reports (resumes on close), and the modal header says **THE MOON** for the lunar set.
- **PinRail top void trimmed** — the ~30vh of empty space above the first card is gone (now a normal small pad); the bottom pad stays so the last card can still scroll up to the "active" band.
- **HUD tidy-ups** — the `DECLASSIFIED · RELEASE_01` stamp no longer touches the top-right corner bracket (extra right padding on the bar); the `TRACKING · N RECORDS` ticker is **green** now instead of red (it clashed sitting under the red stamp); the `← BACK TO GLOBE` chip is removed from the `/about` page (`PageLayout` got a `hideBack` prop).

### Mobile rework + UFOs + autoplay (later 5/9 session)
- **RecordModal is now a draggable bottom sheet on mobile** (`≤767px`). It opens parked partway up the screen (a "peek": ~46% covered, the auto-rotating globe still fully visible *and interactive* above it) showing the location, type badge, record title, agency · year, and a glimpse of the media. A grabber bar + a "↑ SWIPE UP FOR THE FULL REPORT" strip; drag the grabber/header up to expand it to ~92dvh for reading, drag down to collapse back to peek, drag down again (or flick) to dismiss. Tapping a different pin while it's peeked just swaps the content, so you can browse pin titles without losing the globe. The backdrop is click-through in sheet mode (`pointer-events: none`) so the globe behind stays live; only the sheet catches taps. Desktop keeps the existing left-docked panel unchanged. Snap-point logic extracted to a pure, unit-tested helper (`src/lib/bottomSheet.ts`, `tests/bottomSheet.test.ts`).
- **Fixed: tapping a location pin on mobile glitched** (the pin "bounced" up and down and nothing opened). Causes: the `onPointHover` "grow the hovered pin" effect was firing on the synthetic mouse events a tap produces (→ the bounce) — it's now a no-op on touch devices; and the old "first tap = preview bubble, second tap = open" flow plus its re-renders churned the globe layers — replaced with a single tap that opens the record directly (the new bottom sheet *is* the preview). Also removed the floating `TouchPreview` bubble entirely. The Globe accessor props (`pointAltitude` / `pointRadius` / `pointLabel` / `customThreeObject` / `customThreeObjectUpdate` / `onPointHover` / ring color / lat-lng) are now memoized so re-renders (e.g. opening a modal) no longer rebuild every pushpin mesh.
- **Videos autoplay in the modal window** — the `<video>` hero gets `autoPlay playsInline`; the DVIDS `<iframe>` embeds get `allow="autoplay; fullscreen"`. (Opening the modal is a user gesture, so autoplay-with-sound is allowed.)
- **More UFOs, and you can actually catch one** — `UFO_POOL` grew from 5 to 8 saucers (3 new desaturated-metal tones: brushed aluminum / slate steel / graphite, all kept clear of the pin palette and of each other); concurrent cap 2 → 4 and the spawn cool-down 18s → 8s, so there's usually a couple drifting around. Their "flee from the cursor" behaviour is now an *escapable dart*: the spook radius shrank (`FLEE_NDC_RADIUS` 0.18 → 0.12 — only bolts when you're really closing in), the flee impulse is much gentler (kick ×6 → ×2.2, top flee speed 9× cruise → 3×) and shorter (1.1s → 0.65s), and each saucer now carries an invisible ~4.6-unit hit-sphere so a click near it registers — so with a bit of effort you can run one down and open its transmission.
- **Fixed: type-badge layout bug** — the global `.type-badge` rule set `position: absolute; top/left`, which leaked into RecordCard's `right`-anchored badge (stretching "PDF"/"IMG" into a full-width bar across every gallery card) and into RecordModal's flex header (the "DOCUMENT" badge overlapping the location text). The global rule now only carries the visual styling; the thumbnail-overlay surfaces (Hall of Fame cards, Pin Rail cards) opt into the absolute corner positioning explicitly; the modal header and gallery cards get clean inline / corner badges.
- **"UNLOCATED" → "LOCATION UNKNOWN"** in the RecordModal header (matches the `/no-location` page and the dock chip).
- **Hall of Fame chip stays purple on hover** — it was inheriting `.chip`'s green `:hover` colour/border; the `.hof-chip:hover` rule now keeps the `--color-hof` purple (text, border, and the ★ glow).
